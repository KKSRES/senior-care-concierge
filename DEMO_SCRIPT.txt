[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the underlying workflow architecture.
Every input begins at the Security Checkpoint node. This node acts as a gatekeeper, performing PII scrubbing and prompt injection mitigation to ensure complete safety and logging of all requests.
Next is the Orchestrator Agent. The orchestrator acts as the central router, parsing user intent and delegating queries to our three specialized sub-agents: the Medication Agent, the Appointment Agent, or the Resource Agent.
Our sub-agents invoke domain-specific tools exposed by a local stdio-based MCP Server.
If the Appointment Agent drafts a new clinic visit, the workflow routes to the Human Confirmation Node. This is a Human-in-the-Loop checkpoint that pauses execution and asks the user for explicit confirmation before booking.
Finally, once confirmed or completed, the Final Output Node cleans the session state and returns a structured response to the user. (pause)
